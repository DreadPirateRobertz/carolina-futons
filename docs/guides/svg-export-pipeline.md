# SVG Export Pipeline: Figma ‚Üí Optimize ‚Üí Token Inject ‚Üí Wix

## Overview

This pipeline converts Figma-exported SVGs into production-ready assets for the Carolina Futons website. It removes Figma bloat, maps hardcoded colors to brand tokens, and wraps SVGs for Wix HtmlComponent injection.

## Pipeline Steps

```
Figma Export ‚Üí optimizeSvg ‚Üí injectBrandTokens ‚Üí wrapForWixHtmlComponent
     ‚Üì              ‚Üì                ‚Üì                     ‚Üì
  Raw SVG     Clean SVG      Tokenized SVG          Wix HTML file
```

### 1. Export from Figma

- Select your illustration frame/component in Figma
- Right-click ‚Üí Export ‚Üí SVG
- Use settings: Include "id" attribute = OFF, Outline Text = ON
- Save to a working directory

### 2. Run the Pipeline

```bash
node scripts/svgPipeline.js path/to/illustration.svg
```

This produces three files:
- `illustration.optimized.svg` ‚Äî Clean SVG with Figma metadata removed
- `illustration.tokenized.svg` ‚Äî SVG with brand token data-attributes
- `illustration.wix.html` ‚Äî Complete HTML wrapper for Wix HtmlComponent

Optional output directory:
```bash
node scripts/svgPipeline.js input.svg --output ./output/
```

### 3. Integrate into Wix

1. Copy the `.wix.html` content
2. In Wix Studio, add an HtmlComponent to your page
3. Paste the HTML into the component's code
4. Use `$w('#htmlComponent').postMessage(svg)` to update dynamically

## What the Pipeline Does

### Optimization (Step 1)

Removes Figma export bloat:
- XML declarations (`<?xml ...?>`)
- HTML comments (`<!-- Generated by Figma -->`)
- `<metadata>` elements
- Empty `<style>` and `<defs>` blocks
- Figma-specific attributes and namespaces
- Excess whitespace

Typical size reduction: 20-40%.

### Brand Token Injection (Step 2)

Replaces hardcoded hex colors with `data-token-*` attributes:

```xml
<!-- Before -->
<rect fill="#E8D5B7"/>

<!-- After -->
<rect fill="#E8D5B7" data-token-fill="colors.sandBase"/>
```

This preserves visual rendering while documenting which brand token each color maps to. The data-attributes enable future tooling to dynamically swap colors.

### Supported Brand Colors

| Hex | Token | Usage |
|-----|-------|-------|
| `#E8D5B7` | `colors.sandBase` | Primary background |
| `#3A2518` | `colors.espresso` | Primary text/dark |
| `#5B8FA8` | `colors.mountainBlue` | Accent blue |
| `#E8845C` | `colors.sunsetCoral` | CTAs, accent warm |
| `#B8D4E3` | `colors.skyGradientTop` | Sky/decorative |
| `#F0C87A` | `colors.skyGradientBottom` | Sky/decorative |
| `#FAF7F2` | `colors.offWhite` | Light background |

Full list: see `src/public/sharedTokens.js`.

### Wix HTML Wrapper (Step 3)

Wraps the SVG in a responsive HTML document:
- 100% width/height CSS for container-fill
- `postMessage` listener for dynamic SVG updates from Wix page code

## Programmatic Usage

```javascript
import { processSvgPipeline } from './scripts/svgPipeline.js';

const rawSvg = fs.readFileSync('mountain.svg', 'utf8');
const result = processSvgPipeline(rawSvg);

console.log(result.report);
// { originalSize: 2400, optimizedSize: 1600, savingsPercent: 33,
//   replacements: [...], tokenCount: 5 }
```

Individual functions are also exported:
- `optimizeSvg(svg)` ‚Äî Just optimize
- `injectBrandTokens(svg, { report: true })` ‚Äî Just inject tokens
- `wrapForWixHtmlComponent(svg)` ‚Äî Just wrap for Wix
- `buildColorMap()` ‚Äî Get the hex‚Üítoken mapping

## Quality Checklist

Before committing a processed SVG:

- [ ] Pipeline ran without errors
- [ ] Token report shows all brand colors mapped (no unmapped hex in output)
- [ ] SVG renders correctly in browser (open `.optimized.svg`)
- [ ] Wix HTML renders correctly (open `.wix.html` in browser)
- [ ] Visual comparison against `design.jpeg` aesthetic
- [ ] File size is reasonable (< 50KB for most illustrations)
